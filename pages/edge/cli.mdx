# Wasmer deploy CLI

Wasmer provides you with a CLI to deploy and manage your applications on the edge.

## Installation

You can check how to install the Wasmer CLI in the [official docs](/install)

## Usage

Wasmer has two main commands for deploy:

- `wasmer apps`
- `wasmer deploy`

<Callout type="info" emoji="ℹ️">
  <b> Note: </b> You must login to wasmer to use the deploy command.
</Callout>

#### `wasmer app logs`

This command will show you the logs of an app.

```shell
$ wasmer app logs --help
Show an app

Usage: wasmer app logs [OPTIONS] <IDENT>

Arguments:
  <IDENT>
          The name of the app.

          Eg: - name (assumes current user) - namespace/name - namespace/name@version

Options:
      --token <TOKEN>
          [env: WASMER_TOKEN=]

      --registry <REGISTRY>


  -f, --format <FORMAT>
          Output format. (json, text)

          [default: table]

      --from <FROM>
          The date of the earliest log entry.

          Defaults to the last 10 minutes.

      --until <UNTIL>
          The date of the latest log entry

      --max <MAX>
          Maximum log lines to fetch. Defaults to 1000

          [default: 1000]

  -h, --help
          Print help (see a summary with '-h')
```

#### `wasmer app init`

This command is used to initialize an app with a template.

```shell
$ wasmer app init
Initialize apps with templates

Usage: wasmer app init <COMMAND>

Commands:
  static-site  Initialize a static site
  help         Print this message or the help of the given subcommand(s)

Options:
  -h, --help  Print help
```

You can use the `static-site` command to initialize a static site. For a complete guide on how to use this command, check the [static site guide](/deploy/static-site).

### `wasmer deploy`

This command is used to deploy an app. Running this command will automatically publish an app to the registry and then deploy it to the edge.

For a complete guide on deploying an app, check the [deploy guide](/todo/link).

```shell
$ wasmer deploy --help

Deploy apps to the Wasmer Edge

Usage: wasmer deploy [OPTIONS] [PATH]

Arguments:
  [PATH]
          The path to a YAML file with the deploy config

Options:
      --token <TOKEN>
          [env: WASMER_TOKEN=]

      --registry <REGISTRY>


  -f, --format <FORMAT>
          Output format. (json, text)

          [default: yaml]

      --no-validate
          Skip local schema validation

      --publish-package
          Automatically publish the package referenced by this app.

          Only works if the corresponding wasmer.toml is in the same directory.

      --non-interactive
          Do not prompt for user input

  -h, --help
          Print help (see a summary with '-h')
```

The arguments for this command are:

- `--token`: The token to use for authentication. You can also use the `WASMER_TOKEN` environment variable. This defaults to the token in your `~/.wasmer/wasmer.toml` file.
- `--registry`: The registry to use. This defaults to the active registry in your `~/.wasmer/wasmer.toml` file.
- `--format`: The format to use for the output. This defaults to `yaml`.
- `--no-validate`: Skip local schema validation.
- `--publish-package`: Automatically publish the package referenced by this app. Only works if the corresponding `wasmer.toml` is in the same directory.
- `--non-interactive`: Do not prompt for user input.
